tester.sh 는 shell프로그램으로 만들어졌으며, 2차 과제 test를 위해 만들어졌습니다.
모든 input txt 파일에 대해 1~4번 알고리즘 테스트를 실행해줍니다.
테스트 실행 후, 각 input파일에 대해 구현해 놓은 result txt 파일이 생성됩니다.

주의사항
0) 1~4번 알고리즘이 모두 구현되어 있어야 합니다.
  구현되어 있지 않은 알고리즘 제외 방법
	  vi tester.sh 로 편집기를 열어줍니다.
	  1~4 번 알고리즘 중, 구현되지 않은 알고리즘이 있다면 해당 줄을 지워줍니다.

1) 테스트 실행 시, 기존 result 파일은 모두 사라집니다.

2) input 파일은 무조건 _input.txt 로 끝나야 합니다.

3) output 파일은 무조건 result 로 시작해야 합니다.



Happy Hacking!
